Osxdocker ← Back to projects

Categories: personal
Tags: python osx mac cli

I recently published my first python package, osxdocker!

I spend a lot of time looking at docker logs day to day. Unfortunately clearing docker logs on OSX/MAC turns out to be… complicated. The issue stems from docker running on a VM on OSX/MAC. So I built a little python CLI to handle it, with some bash completion and error handling.

Then I decided to try to get it on PyPI. It turned out to be a really interesting process.

  • I converted to using poetry, after being a longtime pipenv user. The main issue it solved was trying to keep pyproject.toml and Pipfile in sync.
  • I learned a lot about PyPI, Sphinx, ReadTheDocs, Github Actions, and continuos delivery.

It’s still not perfect, but it does what I need, so I’m pretty happy with it for now. I hope to add to it as I discover more things that are specifically tricky on OSX.
